# 🤖 HuntTech Protocols Bot

Telegram-бот для извлечения конспектов встреч из почтового ящика (IMAP),
генерации структурированного AI-саммари по шаблону (промпту) и опциональной
публикации в Яндекс Вики.

**Telegram:** [@hunttech_protocols_bot](https://t.me/hunttech_protocols_bot)

---

## 📋 Краткое описание

Бот автоматизирует обработку ежедневных «Конспектов встреч» Совета директоров
IT-компании HUNT TECH:

1. Секретарь высылает текстовую расшифровку встречи на почту в формате `.txt`
2. Бот забирает письма **без снятия флага UNSEEN** (сохраняет непрочитанными)
3. Рекрутер через Telegram выбирает конспект и нажимает 🟢 «Саммари»
4. Нейросеть по заданному шаблону-промпту формирует структурированный протокол
5. Опционально — публикация протокола в Яндекс Вики

**Текущий статус:** проект в активной эксплуатации.

---

## 🚀 Основные возможности

### 📬 Почтовый мониторинг (IMAP)

- Подключение к любому IMAP-серверу (Яндекс, Mail.ru, Google и др.)
- Загрузка писем с вложениями `.txt` без снятия флага `\\Seen`
- Просмотр непрочитанных конспектов (`/list`), всех за 7 дней (`/list all`),
  только новых (без повторов, `/list new`)

### 🧠 AI-саммари

- Поддержка провайдеров: OpenAI, OpenRouter, DeepSeek, Google Gemini, Grok (xAI),
  Anthropic Claude, собственный endpoint
- Выбор модели через меню при настройке
- Генерация по заданному шаблону-промпту (тема конспекта → промпт)
- Результат: структурированный протокол встречи

### 📝 Управление промптами (шаблонами)

- CRUD: создание, просмотр, редактирование, удаление промптов
- Автоматическое сопоставление: промпт привязывается к конспекту,
  если название конспекта начинается с темы промпта (без учёта регистра)

### 📚 Публикация в Яндекс Вики

- Аутентификация через IAM (JWT-токен сервисного аккаунта Yandex.Cloud)
- Три режима публикации:
  - `auto` — автоматически после генерации саммари
  - `button` — кнопка «📤 В Wiki» под каждым саммари
  - `off` — публикация отключена

### 🗄️ База данных (PostgreSQL, опционально)

- Хранение истории обработанных встреч и саммари
- Подключается при наличии переменных `DB_HOST` и `DB_PASSWORD` в `.env`
- Административная настройка через `/setup db`

## 👤 Пользовательские сценарии

### Первоначальная настройка

1. `/start` — приветствие, запуск онбординга
2. `/init` — сброс и пошаговая настройка (4 шага: Email → IMAP-сервер → Логин → Пароль)
3. `/setup ai` — выбор AI-провайдера, ввод API-ключа, выбор модели
4. `/setup wiki` — подключение Яндекс Вики (опционально)

### Повседневное использование

1. `/list` — просмотр непрочитанных конспектов
2. Выбрать конспект → нажать 🟢 «Саммари» (если подходящий промпт найден)
   или 🟡 «Выбрать промпт» (если нужно указать вручную)
3. Получить готовый протокол встречи
4. Опционально: нажать «📤 В Wiki» для публикации

### Административные операции

- `/setup db` — настройка PostgreSQL (5 шагов)
- `/setup show` — просмотр текущих настроек

## 🏗 Архитектура

```text
Telegram (aiogram 3.x)
       │
       ▼
┌─────────────────────────────────────────┐
│          Bot Application (bot.py)         │
│                                           │
│  ┌────────────┐  ┌────────────────────┐   │
│  │  FSM-мастер │  │  HELP registry     │   │
│  │  (11 сост.) │  │  (25+ команд)      │   │
│  └────────────┘  └────────────────────┘   │
│                                           │
│  ┌────────────────────────────────────┐   │
│  │  Integrations (адаптеры)            │   │
│  │  ai_adapter · config_adapter       │   │
│  │  db_adapter · logging_adapter      │   │
│  │  menu_adapter                      │   │
│  └────────────────────────────────────┘   │
└─────────────────────────────────────────┘
       │           │            │
       ▼           ▼            ▼
┌──────────┐ ┌──────────┐ ┌──────────────┐
│   IMAP   │ │    AI    │ │  Yandex Wiki │
│ (почта)  │ │ (LLM)    │ │  (IAM+JWT)   │
└──────────┘ └──────────┘ └──────────────┘
       │
       ▼
┌──────────┐
│PostgreSQL│
│(опционально)│
└──────────┘
```

**Ключевые решения:**
- Асинхронная архитектура (aiogram 3.x)
- Состояние FSM хранится в памяти (`MemoryStorage`)
- Конфигурация пользователей — в JSON-файле (`users.json`)
- Промпты — в JSON-файле (`prompts.json`)
- Использует `hunttech-bot-common` через адаптеры (`integrations/`)
- IMAP-запросы синхронные (imaplib из stdlib)

## 🔧 Установка

```bash

cd tg_conspect_bot

# 2. Установка зависимостей
```

## ⚠️ Типичные проблемы

| Проблема | Диагностика | Решение |
|----------|------------|---------|
| Неверный Telegram-токен | Бот не отвечает | Заменить `TG_TOKEN` в `.env`, перезапустить |
| Почта не подключается | `/setup` — ошибка на шаге проверки | Проверить пароль приложения (не обычный пароль) |
| IMAP-сервер недоступен | Таймаут при подключении | Проверить IMAP-сервер и порт (993 для SSL) |
| AI не отвечает | `/setup ai test` — ошибка | Проверить API-ключ и баланс |
| Модель не найдена | Ошибка AI: 404 | Выбрать доступную модель через `/setup ai` |
| Wiki не публикуется | `/setup wiki test` — ошибка | Проверить SA-ключ, ID организации, права |

## ⛔ Ограничения

- IMAP-запросы синхронные (могут блокировать event loop при большом объёме писем)
- Промпты хранятся в JSON-файле (не в БД)
- Нет webhook — только long polling
- Нет очередей для фоновой обработки
- Нет автоматического расписания проверки почты (только ручной запрос)
