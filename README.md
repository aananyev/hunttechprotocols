# 🤖 HuntTech Protocols Bot

Telegram-бот для извлечения конспектов встреч из почты и генерации AI-саммари.

**Бот:** [@hunttech_protocols_bot](https://t.me/hunttech_protocols_bot)

---

## 📋 Бизнес-назначение

Бот автоматизирует рутину Совета директоров IT-компании HUNT TECH:

- Каждое утро после ежедневного совещания секретарь высылает **текстовую расшифровку** встречи в виде `.txt`-файла на почту с темой «Конспект встречи»
- Бот забирает эти письма **не помечая непрочитанными** (UNSEEN сохраняется)
- Под каждым конспектом — кнопка, которая отправляет расшифровку в нейросеть
- Нейросеть по заданному **промпту-шаблону** формирует структурированное саммари

### Пример бизнес-процесса

```
Секретарь → отправляет "Конспект встречи от 01.01" с .txt-файлом
        ↓
Рекрутер → /list → видит конспект → нажимает 🟢 Саммари
        ↓
Нейросеть → формирует отчёт по шаблону Совета директоров
        ↓
Рекрутер → получает готовый протокол встречи
```

## 🚀 Возможности

### 📬 Почта (IMAP)

| Команда | Описание |
|---------|---------|
| `/setup` | Настройка подключения к почте (4 шага) |
| `/init` | Сброс и настройка заново |
| `/list` или `/конспекты` | Только **непрочитанные** конспекты (UNSEEN НЕ снимается) |
| `/list_all` или `/все_конспекты` | Все конспекты за 7 дней |

Поддерживаются: **Яндекс**, **Mail.ru**, **Google** и любой IMAP-сервер.

### 🤖 AI-саммари (одним нажатием)

- Поддержка **OpenRouter**, **OpenAI**, **DeepSeek**, **свой endpoint**
- При каждом конспекте — 🟢 зелёная кнопка (есть подходящий промпт) или 🟡 жёлтая (нужно создать)
- Нейросеть получает **системный промпт** (шаблон отчёта) + **текст конспекта**
- Настройка: `/setup_ai`

### 📝 Управление промптами (шаблонами для AI)

| Команда | Описание |
|---------|---------|
| `/prompt` или `/промпты` | Список всех промптов + кнопки |
| `/add_prompt` или `/prompt_add` | Добавить промпт (тема → текст шаблона) |
| `/edit_prompt 1` или `/prompt_edit` | Редактировать по номеру или из списка |
| `/text_prompt 2` или `/prompt_text` | Показать текст промпта |
| `/delete_prompt` или `/prompt_delete` | Удалить промпт |

**Бизнес-правило сопоставления:** промпт автоматически привязывается к конспекту, если название конспекта **начинается** с темы промпта (без учёта регистра).

### 🔧 Система

| Команда | Описание |
|---------|---------|
| `/help` или `/помощь` | Полная справка |
| `/start` | Приветствие + авто-онбординг |
| `/setup_ai` или `/setup_llm` | Настройка AI-провайдера |

## 🏗 Архитектура и стек

| Компонент | Технология |
|-----------|-----------|
| Telegram API | aiogram 3.x (асинхронный фреймворк) |
| Почта | IMAP (imaplib, stdlib) — BODY.PEEK[] + явное снятие флага \Seen |
| Нейросеть | OpenAI-compatible API (httpx) |
| Хранилище | JSON-файлы (многопользовательские) |
| FSM-диалоги | aiogram.fsm (StatesGroup) |
| Конфигурация | python-dotenv (TG_TOKEN в .env) |

### Структура файлов

```
tg_conspect_bot/
├── bot.py            # Основной код (все хендлеры + IMAP + AI)
├── .env              # TG_TOKEN (не в git!)
├── .env.example      # Шаблон для новых разработчиков
├── requirements.txt   # Зависимости
├── .gitignore        # Исключает секреты
├── users.json        # Учётные данные пользователей (не в git!)
├── prompts.json      # Сохранённые промпты
└── notes_cache.json  # Кеш конспектов для кнопки Саммари (не в git!)
```

## ⚙️ Установка и запуск

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Создать .env из шаблона и вписать токен
cp .env.example .env
# Отредактировать .env — вставить TG_TOKEN от @BotFather

# 3. Запустить
python3 bot.py
```

- **AI-ключи** — в `users.json`, исключён из git
- **Пароли приложений** Яндекса — не хранятся в git
- **Кеш конспектов** — в `notes_cache.json`, исключён из git
